Documentação da API REST

18 endpoints do backend Node.js + Express — Sistema de Reservas CCHLA/UFRN

Base URL: /api

Autenticação: Bearer token JWT nas rotas protegidas. Token obtido via /api/fazerLogin, válido por 24h.

Rotas Públicas

POST /api/checarDisponibilidade Público

Descrição: Verifica conflitos de horário entre reservas solicitadas e reservas existentes no banco.

Request:

{
  "reservas": [
    { "auditorio": "Auditorio A", "dateStr": "2026-07-20", "hIni": "08:00", "hFim": "10:00" }
  ]
}

Response 200:

{
  "conflitos": [
    { "solicitado": {...}, "conflitoCom": "Nome da Atividade", "dataBr": "20/07/2026", "hIniBanco": "08:00", "hFimBanco": "10:00" }
  ],
  "agendaCompleta": [...]
}

Validações: Ignora reservas com status "Reprovado" ou "Cancelado". Busca reservas dos últimos 2 dias até o futuro.

Controller: backend/controllers/reservaController.js:58

POST /api/enviarSolicitacao Público

Descrição: Envia uma solicitação de reserva. Verifica reCAPTCHA, valida dados, insere no MongoDB e dispara e-mail de confirmação.

Request:

{
  "solicitante": "Nome",
  "matricula": "20200000001",
  "email": "email@ufrn.br",
  "telefone": "84999999999",
  "departamento": "DEPT",
  "atividade": "Aula",
  "recaptchaToken": "token_google",
  "reservas": [
    { "auditorio": "Auditorio A", "dateStr": "2026-07-20", "hIni": "08:00", "hFim": "10:00", "teveConflito": false }
  ]
}

Response 200: { "sucesso": true, "ticketGerado": "ABC123" }

Validações:

  • reCAPTCHA obrigatório (verificação real apenas se RECAPTCHA_SECRET configurada)
  • Máximo 50 reservas por solicitação (anti-flood)
  • Campos obrigatórios: auditorio, dateStr, hIni, hFim
  • hIni deve ser anterior a hFim
  • Horários dentro de 07:00–21:30

Controller: backend/controllers/reservaController.js:67

GET /api/obterEventosCalendario Público

Controller: backend/controllers/reservaController.js:162

POST /api/consultarTicket Público

Descrição: Consulta todas as reservas associadas a um ticket.

Response 200: Array de reservas com id, Ticket, Status, Auditório, Atividade, Início, Fim, Solicitante, Departamento, Telefone, Email, Data_Solicitacao, Motivo_Reprovacao.

Controller: backend/controllers/reservaController.js:185

POST /api/fazerLogin Público

Descrição: Autenticação de administradores. Retorna token JWT válido por 24h.

Request: { "email": "admin@cchla.ufrn.br", "senha": "******" }

Response 200: { "sucesso": true, "token": "eyJhbG..." }

Response 401: { "erro": true, "mensagem": "E-mail ou senha incorretos." }

Segurança:

  • Rate limit: 5 tentativas falhas em 15 minutos bloqueiam o acesso (exceto root principal)
  • Log de segurança: todas as tentativas registradas no SecurityLog
  • Senha verificada com bcrypt
  • Token JWT assinado com JWT_SECRET, expira em 24h

Controller: backend/controllers/authController.js:6
