server.js — Ponto de Entrada

Arquivo: backend/server.js (108 linhas)

Configura e inicializa o servidor Express na porta 3000.

Inicialização

  1. dotenv — carrega variáveis de ambiente do .env
  2. CORS — permite apenas origens autorizadas (https://reservas.cchla.ufrn.br e http://localhost)
  3. JSON body parser — express.json()
  4. MongoDB — conexão via Mongoose com MONGO_URI
  5. Middleware de segurança global — rastreia acessos a rotas específicas conforme configurado no SecurityConfig

Middleware de Rastreamento

Middleare global (server.js:42) que intercepta requisições e registra no SecurityLog se o monitoramento estiver ativo para aquela rota. Ignora a rota de login (já rastreada no authController).

Middleware

auth.js

Arquivo: backend/middleware/auth.js (48 linhas)

Middleware JWT obrigatório para todas as rotas protegidas. Verifica token Bearer no header Authorization, decodifica com jwt.verify e injeta req.usuarioLogado. Também exporta verificarPermissao para controle granular por ação.

const verificarToken = (req, res, next) => {
  const authHeader = req.headers.authorization;
  if (!authHeader || !authHeader.startsWith('Bearer '))
    return res.status(401).json({ erro: true, mensagem: "Token ausente." });

  const token = authHeader.split('Bearer ')[1];
  const decoded = jwt.verify(token, process.env.JWT_SECRET);
  req.usuarioLogado = decoded;
  next();
};
