Documentação do Frontend

5 SPAs que compõem a interface do Sistema de Reservas de Auditórios do CCHLA/UFRN

1. app-calendario — Calendário de Ocupação

Arquivo: app-calendario/index.html (501 linhas)

Rota: / (raiz) — Página principal do sistema.

Calendário interativo com FullCalendar 6 que exibe todas as reservas aprovadas nos 5 auditórios.

1.1 Funcionalidades

Calendário FullCalendar

Visões: Mês, Semana, Dia e Lista. Navegação entre meses com botões ou dropdown.

Legenda por Auditório

Filtro por auditório (A ao E) com cores distintas. Clique para filtrar eventos.

Tooltip ao pairar

Ao passar o mouse sobre um evento, exibe detalhes: atividade, auditório, horário, solicitante, departamento.

Modal de detalhes

Ao clicar em um evento, abre modal com informações completas. Fecha ao clicar fora ou no X.

Feriados e Manutenções

Eventos especiais com destaque laranja (feriado) ou vermelho (manutenção), exibidos como dia inteiro.

Menu de Solicitações

Dropdown com links para: Solicitar Reserva, Acompanhar Solicitação e Gestão (Secretaria).

Botão "Hoje"

Ao clicar no título do período, retorna para o dia atual.

Responsivo

Layout adaptável para mobile com breakpoints.

1.2 API utilizada

EndpointMétodoDescrição
/api/obterEventosCalendarioGETRetorna todas as reservas com status "Aprovado"

2. app-formulario — Solicitação de Reserva

Arquivo: app-formulario/index.html (1032 linhas)

Rota: /formulario — Página pública para solicitar reservas.

2.1 Funcionalidades

Dados do Solicitante

Campos: nome, e-mail, matrícula, telefone, departamento, atividade.

Seleção de Auditórios

Checkbox para selecionar 1 ou mais dos 5 auditórios disponíveis.

Calendário Semanal

Seleção de data com calendário visual por dia da semana. Fins de semana bloqueados.

Filtro de Horários

Dropdowns de hora início/fim por dia. Se data = hoje, horários passados são desabilitados automaticamente.

Verificação de Conflitos

Ao clicar em "Adicionar à Lista", consulta API de disponibilidade. Exibe alerta rico com conflitos encontrados (auditório, horário, atividade conflitante).

Lista de Reservas

Tabela com itens adicionados, permite remover itens, exibe conflitos.

reCAPTCHA

Proteção contra bots com Google reCAPTCHA.

Envio e Ticket

Ao enviar, exibe tela de sucesso com ticket gerado, dados da reserva e botões de imprimir/WhatsApp.

Histórico de Solicitações

Busca automática por e-mail/matrícula: se já solicitou antes, reusa o ticket e exibe histórico.

E-mail de Confirmação

Após envio bem-sucedido, dispara e-mail automático para o solicitante com resumo e link de acompanhamento.

Validação de Horário

Bloqueia horário de término menor/igual ao início. Limite operacional 07:00–21:30. Máximo 50 solicitações por vez.

2.2 APIs utilizadas

EndpointMétodoDescrição
/api/checarDisponibilidadePOSTVerifica conflitos de horário antes de adicionar à lista
/api/enviarSolicitacaoPOSTEnvia solicitação com dados + reCAPTCHA token

3. app-acompanhamento — Acompanhamento via Ticket

Arquivo: app-acompanhamento/index.html (410 linhas)

Rota: /acompanhamento — Página pública para acompanhar solicitações.

3.1 Funcionalidades

Busca por Ticket

Campo de busca com uppercase automático. Enter ou clique em "Consultar". Link direto via query string ?ticket=XXX.

Dados do Solicitante

Exibe nome, matrícula, departamento, telefone, e-mail e ticket em destaque.

Tabela de Reservas

Lista todas as solicitações associadas ao ticket com data, status, auditório, atividade e início. Paginação de 10 itens.

Status com Cores

Badges coloridos: amarelo (pendente), verde (aprovada), vermelho (reprovado). Tooltip com motivo ao passar sobre "Reprovado".

Impressão

Botão "Imprimir Comprovante" com CSS @media print otimizado. Oculta elementos de navegação.

Contato Secretaria

Link para WhatsApp da secretaria.

Inatividade

Timer de 30 minutos: recarrega a página se o usuário ficar inativo.

3.2 API utilizada

EndpointMétodoDescrição
/api/consultarTicketPOSTRetorna todas as reservas de um ticket

4. app-painel — Painel Administrativo (Secretaria)

Arquivos: app-painel/painel.html (1366 linhas), cad-reservas-secretaria.html, feriados.html, gerir-contas.html, cchlafense.html

Rota: /admin — Requer autenticação JWT.

4.1 painel.html — Gestão Principal de Reservas

Cards de Reserva

Cada reserva exibida em card com header colorido por status (verde/aprovado, amarelo/pendente, vermelho/reprovado).

Filtros Avançados

Filtros por: nome, matrícula, data, status, auditório, departamento, atividade, período e busca geral verde.

CRUD Completo

Aprovar, reprovar (com motivo), editar dados, excluir (root apenas), cadastrar nova reserva manualmente.

Notificação por E-mail

Ao alterar status, botão "Notificar" dispara e-mail automático ao solicitante com o novo status.

Exportação

Exportar dados filtrados para CSV, 1 por linha ou agrupado por ticket.

Páginação

Controles de página e seleção de itens por página.

Modal de Inatividade

Após período de inatividade, exibe modal "Manter-se logado" ou faz logout automático.

Menu Dropdown

Navegação para: Cadastro Manual, Feriados, Segurança (CCHLAfense), Usuários, Alterar Senha.

4.2 cad-reservas-secretaria.html — Cadastro Manual

Formulário idêntico ao público, porém sem reCAPTCHA. Permite à secretaria cadastrar reservas em nome de terceiros. Inclui verificação de conflitos e e-mail de confirmação.

4.3 feriados.html — Gestão de Feriados e Manutenções

CRUD de Eventos

Cadastrar, editar e excluir feriados e manutenções. Campos: tipo, descrição, data, horário.

Filtros

Filtrar por tipo (feriado/manutenção), mês e busca textual. Exibição em tabela.

Cores no Calendário

Feriados em laranja, manutenções em vermelho escuro. Aparecem no calendário público.

4.4 gerir-contas.html — Gestão de Usuários (Root)

CRUD de administradores. Apenas usuários root podem acessar. Criação com níveis (admin/root), edição de senha, exclusão (não pode excluir a si mesmo).

4.5 cchlafense.html — Módulo de Segurança

Inspirado no Wordfence. Exibe estatísticas com gráficos Chart.js:

  • Top IPs bloqueados — IPs com mais tentativas de login falhas
  • Top usuários com falha — E-mails com mais tentativas incorretas
  • Falhas diárias — Gráfico de barras dos últimos 14 dias
  • Configuração de rastreamento — Checkboxes para ativar/desativar monitoramento por área (calendário, formulário, painel, etc.)

5. app-manutencao — Página de Manutenção

Arquivo: app-manutencao/index.html (200 linhas)

Rota: Usada quando o sistema está fora do ar ou como página 404.

Layout centralizado com logo, mensagem "Em Construção", ícones de redes sociais (Instagram, WhatsApp, E-mail) e informações da Gerência de Redes.

6. Infraestrutura Frontend

6.1 Nginx

Arquivo: nginx.conf (58 linhas)

  • Proxy reverso: /api/ → backend na porta 3000
  • Proxy restrito: /db-admin/ → mongo-express (apenas rede local e 10.0.0.0/8)
  • Rotas das SPAs: / → calendário, /formulario → formulário, /acompanhamento → acompanhamento, /admin → painel
  • Real IP: set_real_ip_from para capturar IP real atrás do proxy

6.2 Docker

Arquivo: frontend.Dockerfile — baseado em nginx:alpine, copia todas as SPAs e o nginx.conf.
